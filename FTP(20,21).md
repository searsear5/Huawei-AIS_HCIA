# FTP (File Transfer Protocol)

## FTP Active Mode vs Passive Mode

### ปัญหาที่ต้องแก้
FTP ใช้ 2 connections:
1. **Control Connection (Port 21)** - สำหรับส่งคำสั่ง
2. **Data Connection (Port 20)** - สำหรับส่งไฟล์จริงๆ

ปัญหาคือ **ใครเป็นคนเริ่ม data connection?**

---

## Active Mode (โหมดเก่า - มีปัญหากับ Firewall)

### วิธีทำงาน:
1. **Client เชื่อมต่อ** Server ที่ port 21 (control)
2. Client บอก Server ว่า "ฉันเปิด port 1234 รอส่งไฟล์"
3. **Server เชื่อมต่อมาหา** Client ที่ port 1234 (data)
4. ส่งไฟล์

### ปัญหา:
❌ **Firewall ของ Client มักบลอก** เพราะ Server พยายามเชื่อมต่อเข้ามา
❌ Client ที่อยู่หลัง NAT/Router มีปัญหา เพราะ Server หา Client ไม่เจอ

```
Client -----> Server (port 21) ส่งคำสั่ง ✅
Client <----- Server (port 20) Server เริ่ม data connection ❌ โดน Firewall บลอก
```

---

## Passive Mode (โหมดใหม่ - แนะนำ)

### วิธีทำงาน:
1. **Client เชื่อมต่อ** Server ที่ port 21 (control)
2. Client บอก "ใช้ passive mode"
3. **Server บอก** Client ว่า "ฉันเปิด port 5678 รอนะ"
4. **Client เชื่อมต่อไป** หา Server ที่ port 5678 (data)
5. ส่งไฟล์

### ข้อดี:
✅ **Client เป็นคนเริ่ม connection ทั้งคู่** → ไม่โดน Firewall บลอก
✅ ทำงานได้ดีกับ NAT/Router
✅ เว็บเบราว์เซอร์ใช้ passive mode เป็นค่าเริ่มต้น

```
Client -----> Server (port 21) ส่งคำสั่ง ✅
Client -----> Server (port 5678) Client เริ่ม data connection ✅ ผ่าน Firewall ได้
```

---

## สรุปแบบง่ายสุด

| Mode | ใครเริ่ม Data Connection | ผลลัพธ์ |
|------|--------------------------|---------|
| **Active** | Server → Client | ❌ มักโดน Firewall บลอก |
| **Passive** | Client → Server | ✅ ใช้งานได้เกือบทุกกรณี |

### จำง่ายๆ:
- **Active Mode**: Server **โทรกลับ** หา Client (มักโดนปฏิเสธ)
- **Passive Mode**: Client **โทรไป** หา Server ทั้ง 2 ครั้ง (ผ่านได้เสมอ)

### Port ที่ใช้:
- **Control**: Port 21 (ทั้ง 2 mode)
- **Data Active**: Port 20 (Server → Client)
- **Data Passive**: Port random สูงกว่า 1023 (Client → Server)
